# ADR-0012 — v0.5.0 tensor-product 2D extension (`Strang2D` + `AxisLift`, separable operators)

**Status**: Accepted
**Date**: 2026-05-01
**Authors**: ai-solutions-architect
**Supersedes**: none. Implements Phase 1 of the approved Path A plan (`/home/volk/.claude/plans/splendid-baking-sedgewick.md`).
**Cross-refs**: ADR-0006 (Strang symmetrization, palindromic order-2 lift), ADR-0008 (ζ-A self-adjoint diffusion), ADR-0011 (TruncatedExp integrator), `contracts/semiflow-core.math.md` §10 (tensor-product theorem) and §9.7 forward-reference at lines 1463-1464 (now anchored to §10), `contracts/semiflow-core.tensor.yaml` (struct schema).

Adopt a **tensor-product 2D extension** for separable generators $L = L_x \otimes I + I \otimes L_y$ via three additions: row-major single-`Vec<f64>` storage `Grid2D`/`GridFn2D` with linear index $\mathrm{idx}(i,j) = j\,n_x + i$ (x is the fast axis); a per-axis adapter `AxisLift<C>` parameterised by `Axis::{X, Y}` that lifts any 1D `ChernoffFunction<S = GridFn1D>` (e.g. `DiffusionChernoff` or `TruncatedExpDiffusionChernoff`) to act on `GridFn2D` row-by-row (X) or column-by-column (Y) reusing the existing 1D kernel verbatim; and a dedicated `Strang2D<X, Y>` Chernoff function implementing the palindromic composition $\Phi^{2D}(\tau) = \mathrm{AxisLift}_X(\tau/2) \circ \mathrm{AxisLift}_Y(\tau) \circ \mathrm{AxisLift}_X(\tau/2)$. The order-2 lift is justified by the commutator identity $[L_x \otimes I,\, I \otimes L_y] = 0$ for separable $L$, which makes the BCH expansion of $\exp(\tau L)$ collapse to the product $\exp(\tau L_x) \cdot \exp(\tau L_y)$ globally — palindromic Strang along axes therefore inherits the existing 1D order-2 result per axis with no additional commutator cancellation required (`math.md` §10 Theorem 7). Per-axis boundary policies remain independent: each `Grid1D` carries its own `BoundaryPolicy`, and CFL checks (TruncatedExp K=4 path, ADR-0011) stay 1D per axis since `AxisLift` invokes the 1D `apply` per row/column with the axis-local `dx`. Rejected alternatives: (a) genuine 2D 5-point Chernoff stencil for non-separable cross-derivatives ($\partial_x\partial_y$) — deferred to v0.7+ behind a separate ADR; correctly handling $[L_x, L_y] \neq 0$ requires a different math (Strang + commutator BCH residual) and is out of scope; (b) `Vec<Vec<f64>>` row-of-rows layout — pointer-chase, $n_y + 1$ allocations on `zeroed_like`, breaks cache locality, violates suckless single-allocation storage; (c) generalising the existing `StrangSplit<D, R>` to be `S`-generic over arbitrary state types — touches stable v0.2.0+ 1D code, scope creep, defeats the contract-stability mandate. Consequences: exactly +5 public types (`Axis`, `AxisLift`, `Grid2D`, `GridFn2D`, `Strang2D`), +0 dependencies, no `SemiflowError` variants added (shape mismatches use `DomainViolation`), no 1D regression risk (1D paths untouched), TruncatedExp K=4 envelope (σ < 0.70 ∧ σ·T·β < 0.80, no `lam_peak` cap; H2 cell exempt) carries over per axis, and the existing forward-reference `math.md:1463-1464` (`$\Phi^{2D}$ inherits the same per-axis order-2 result`) is now anchored to the normative §10 Theorem 7.
