# ADR-0164 — Complex spectral symbol lifts drift `b≠0` into the TT curse-escape (S³ POC)

**Status:** PROPOSED (proof-of-concept; design-only this round) — **AMENDED post-audit (Amendment 1, 2026-06-12).** **Date:** 2026-06-12.
**Branch:** `experiment/triz-s3-curse-escape`. **Builds on:** ADR-0162 (v9.1 real spectral pair-factor).
**Design:** `.dev-docs/specs/s3-triz-general-curse-escape.md` (§6.5–§6.8). **Probes:** `.dev-docs/specs/probe_s3_drift_spectral.py` + `.dev-docs/specs/probe_s3_honest_curse_escape.py`.

**Decision.** Resolve the loudest open S³ fail-loud case — the TT carrier rejecting drift `b≠0` at construction ("symmetric mean-preserving shift, advection unimplemented") — by widening the v9.1 §11.3 *real* spectral pair-factor symbol to a **complex** symbol `exp(τ(a·σ_D2(k) + b·σ_D1(k) + cross))`, where the diffusion is its real part `a·σ_D2=(2cosω−2)/dx²` and the drift is its imaginary part `b·σ_D1=b·i·sinω/dx` (a unit-modulus phase = exact Fourier translation by `bτ`). Because `a,b,r` are real the symbol is conjugate-even, so `ifft(exp(τ·symbol)⊙fft(real))` stays real (TRIZ separation-in-structure: complex per-mode multiplier, real rank-O(1) operator). The make-or-break probe confirms: spectral ≡ independent dense `expm` to **1e-16** with `b≠0` sub-grid present, output real (imag-residue ~1e-16), **no linear solver** (FFT + elementwise complex multiply + real-part take; Theorem-6 R2 honoured).

**Amendment 1 (honest curse-escape — supersedes the original rank claim).** A skeptical adversarial audit returned **VERDICT: OVERCLAIM** on the original curse-escape gate ("coupled+drift evolved-state rank **constant=3 for d=3..6**"). That gate is **vacuous**: the rank is an SVD-tolerance knife-edge (no spectral gap), the smooth IC is low-rank pre-evolution and stays low-rank with zero drift/coupling (an IC-confound), and a generic input evolves to full rank (no compression of generic states — info-theoretically unavoidable). The exactness/drift core (asserts 1,2,3,5,6,7) **survives**; only the rank claim is retracted. It is **replaced** by two honest, tolerance-robust gates (contract §2 assert 4a+4b): **(4a) Δrank-PRESERVATION** — `rank_{b≠0}(eps) − rank_{b=0}(eps) = 0` for ALL eps∈{1e-8…1e-14} and ALL d∈{3,4,5,6}, for BOTH smooth and generic ICs, because per-axis drift is a **unit-modulus separable phase = rank-1 TT operator** (`sv₂/sv₁=8.2e-17`) that cannot increase rank; a difference at fixed eps cancels the knife-edge, and the generic-IC arm (`Δrank=0` even at full rank `n`) defeats the IC-confound; **(4b) OPERATIONAL cost-scaling** — the evolver runs at d=8,10 in `O(d·n·r²)` while the dense `expm` reference is un-formable there (>1 TB), an absolute tolerance-free escape. Paired with a **load-bearing** assert (drift moves the state 5–45%), so Δrank=0 means "drift costs zero rank," not "drift does nothing."

**Honest S³ claim (the only one proven):** the v9.1 constant-coef adjacent-pair curse-escaping carrier extends to **non-zero drift `b≠0`** at **provably zero TT-rank cost**, exactly (≤1e-12 vs independent dense `expm`) and solver-free, running where the dense reference OOMs. **NOT** claimed: absolute generic-input curse-escape (false — no TT method compresses random tensors). **Reconciliation with v9.1:** v9.1's `b=0` scheme shipped the same tolerance-soft `[5,5,5,5]` absolute headline; the Δrank framing is robust regardless, being a difference relative to that exact baseline. **Out of scope (deferred):** variable-coef, non-adjacent/dense coupling, nonlinear, public API. The POC ships ONE evolver variant (`DriftSpectralPairFactor`) + ONE gate (`G_S3_DRIFT_SPECTRAL`: exactness ≤1e-12 vs `expm` + Δrank-preservation sweep + operational cost-scaling; no slope gate — exact scheme), no new dep (reuses `tt_spectral` DFT). Honest scope, audit-satisfying boundary, and FAIL-LOUD list per design §6.5–§6.8 + §8.
