# ADR-0038 — State trait experimental marker for v1.0.0

**Status**: Proposed 2026-05-19 for v1.0.0.

**Context**: Iter-3 bench (`BENCHMARKS-STATUS.md`) showed RC's H-MEM 8/8 SUPPORT and F11 cardiac SUPPORT as flagship strengths, but post-bench architectural analysis revealed two structural facts: (a) the `State<F>` trait introduced in early development (`crates/semiflow-core/src/state.rs`, 70 LoC, 4 methods: `axpy`, `scale`, `norm_sup`, `zeroed_like`) is **bypassed by all 12 leaf ChernoffFunction impls** via direct `.values[i]` indexing — the trait promise is architecturally void; (b) the H-MEM Pareto dominance is a **measured property** of concrete `Vec<f64>`-backed `GridFnXD` types, NOT a contractual guarantee enforced by trait signatures. Per researcher analysis (2026-05-19) and user strategic direction, full Trait State refactor is the right v2.0.0 work but premature for v1.0.0 (3-layer trait shape `State` / `HilbertState` / `Discrete` requires 48h spike validation per ADR §"Out of scope").

**Decision**: Ship v1.0.0 with current concrete `Grid1D/2D/3D + GridFnXD` API frozen per ADR-0035, but mark `State<F>` trait and `ChernoffFunction::type S` as `#[doc(hidden)]` experimental — public symbols semver-protected through v1.x, with explicit "may change in v2.0.0" annotation in rustdoc. Add `#![doc = "experimental"]` to `state.rs` module docs. The 48h graph-Laplacian validation spike (separate task, `crates/remizov-graph-spike/` — not published) confirms 3-trait shape before v2.0.0 design freeze. v1.0.0 README documents H-MEM as measured property of concrete types, not contractual invariant. v2.0.0 then promotes State/HilbertState/Discrete to central abstraction with full Vector 3 (anisotropic angle) + Vector 4a (Adjoint-Chernoff) + graph/manifold PDE support, breaking API minimally via the trait surface.

**Consequences**: v1.0.0 ships on schedule with FFI/PyO3/WASM commitment from v0.10.0-v0.11.0 intact. Trait State refactor work is NOT cancelled — it becomes v2.0.0 milestone with researcher recommendation validated by spike before architectural lock-in. H-MEM brand positioning becomes defensible scientific claim post-v2.0.0 (when trait Required: `axpy(&mut self, ...)` signals in-place — but per Rust mechanics, no compile-time no-alloc enforcement; need `cap-allocator` test harness + `MatMut`-style split for true invariant, evaluated in spike). Wave A2 (Strang3D serial scratch, commit `52bac39`) is compatible with future trait refactor — scratch pattern propagates cleanly to State::axpy_inplace contract. Wave B/C/D (v0.13.0 perf milestone) continue as planned; SIMD on TruncatedExp4 (Wave B3) gets specialised `<f64>` fast path retained at v2.0.0 per ADR-0019 Amendment 2 pattern. Citation: researcher memo "Trait State / Banach-space generic core" 2026-05-19.
