# ADR-0118 — Higher-order divergence-form stencils for ∂_x(a ∂_x ·) (v7.0.0 KEYSTONE stencil)

- **Status**: ACCEPTED 2026-06-05 (machinery sound; both stencils sympy-verified) — the **6th-order** stencil is the one the ζ⁶/ζ⁸ TRUTHFUL_ORDER gate uses (ADR-0119 GO at N=4096/T=10); the 4th-order stencil is retained as a lighter additive option.
- **Decision-maker**: ai-solutions-architect
- **Type**: ADDITIVE — `pub(crate)` helpers `apply_div_form_4th` and `apply_div_form_6th`; no public surface change. Existing 3-point `apply_div_form` retained for callers that want the conservative 2nd-order operator.
- **Authority for**: math.md §40.7 (NEW higher-order divergence stencil subsection).
- **PRE-FLIGHT**: `scripts/verify_div_stencil_4th.py` — `T_DIV_STENCIL_4TH` 4/4 PASS 2026-06-05; `scripts/verify_div_stencil_6th.py` — `T_DIV_STENCIL_6TH` 4/4 PASS 2026-06-05.

**Decision.** Replace the 2nd-order 3-point flux `(Af)_i = [a_{i+½}(f_{i+1}−f_i) − a_{i-½}(f_i−f_{i-1})]/dx²` (the spatial-floor bug diagnosed in ADR-0110 AMENDMENT 1 Error 2, `diffusion4_zeta4.rs:331`) with higher-order flux-difference discretisations of `∂_x(a ∂_x f)`, evaluating `a` EXACTLY at the half-nodes `x_{i±½}` (known coefficient, no FD error). Two staggered stencils are provided:

- **4th order** (`apply_div_form_4th`, 5-point): half-node first derivative and outer flux divergence both use the staggered weights `(1, −27, 27, −1)/(24 dx)`. Sympy (`T_DIV_STENCIL_4TH`) proves O(dx⁴): half-node derivative O(dx⁴); const-a composition = 4th-order Laplacian; variable-a truncation dx² AND dx³ coefficients vanish; generating polynomial `1−27z+27z²−z³` divisible by `(1−z)` → conservation form. Drops the spatial floor `T·dx² → T·dx⁴`.

- **6th order** (`apply_div_form_6th`, 7-point — THE GATE STENCIL): half-node first derivative and outer flux divergence both use the staggered weights `(−9, 125, −2250, 2250, −125, 9)/(1920 dx)` on offsets `{∓5/2, ∓3/2, ∓1/2}·dx`. Sympy (`T_DIV_STENCIL_6TH`) proves O(dx⁶): half-node derivative O(dx⁶); const-a composition = 6th-order Laplacian; variable-a truncation dx², dx³, dx⁴ AND dx⁵ coefficients all identically zero as symbolic functions of `a, a', …, f, f', …`; generating polynomial `−9+125z−2250z²+2250z³−125z⁴+9z⁵` divisible by `(1−z)` → conservation form (Neumann mass preserved). Drops the spatial floor `T·dx⁴ → T·dx⁶`.

**Why the 6th-order stencil is the gate stencil.** ADR-0119's decisive oracle shows that the 4th-order `T·dx⁴` floor alone does NOT open the ζ⁸ pair-slope window at the v6.0.0 horizon (N=512); even at the gate horizon N=4096/T=10 the 4th-order stencil clears the gate only marginally (ζ⁸ −7.9835) and its τ-window is thin. The **6th-order** `T·dx⁶` floor (≈ 2.1e-13·T at N=4096, ~5 orders below the temporal signal at every ladder point) gives a robust non-empty window for BOTH K=6 and K=8 with the finest pair (8→16) NOT floor-dominated → ADR-0119 GO. Because `diffusion4_zeta4.rs` is at 633 LoC (near the watched 700-LoC architectural cap), both higher-order stencils + their K-jet builders land in a NEW module `crates/semiflow-core/src/diffusion4_zeta4_stencil_ho.rs`, not by inflating the existing file. The 4th-order stencil remains a genuine accuracy improvement and a lighter additive option for callers who do not need the ζ⁶/ζ⁸ gate horizon.
