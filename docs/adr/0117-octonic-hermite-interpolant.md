# ADR-0117 — OCTONIC-Hermite degree-9 virtual-node interpolant (v7.0.0 KEYSTONE sampler)

- **Status**: ACCEPTED 2026-06-05 (machinery sound; sympy 5/5 PASS) — but see ADR-0119 for the gate it was meant to unblock (NO-GO, honest-defer).
- **Decision-maker**: ai-solutions-architect
- **Type**: ADDITIVE — new `InterpKind::OctonicHermite` enum variant; default UNCHANGED (stays `SepticHermite`). No BREAKING surface.
- **Authority for**: math.md §41.bis (NEW). Sibling of ADR-0109 (SepticHermite §40).
- **PRE-FLIGHT**: `scripts/verify_octonic_hermite_weights.py` — `T_OCTONIC_HERMITE` 5/5 PASS 2026-06-05.

**Decision.** Introduce `InterpKind::OctonicHermite`, a degree-9 Hermite virtual-node interpolant matching `f, f', f'', f''', f''''` (FIVE data per node) at both cell endpoints — 5×2 = 10 constraints → unique degree-9 polynomial with leading residue `O(dx¹⁰)` (Birkhoff-Garabedian-Lorentz 1983). Ghost derivatives are computed by 10-point central Fornberg FD weights (`f'`: `(−1, 5, −60, 420, −840, 840, −420, 60, −5, 1)/(840 dx)` on `{∓5..∓1}`; `f''`,`f''''` via 9-point symmetric, `f'''` via 10-point antisymmetric). Value-weight closed form `a₀(s) = (1−s)⁵·(1 + 5s + 15s² + 35s³ + 70s⁴)` (degree 9, all 10 endpoint constraints zero-residual). New file `crates/semiflow-core/src/grid_chebyshev_octonic.rs` mirrors `grid_chebyshev_septic.rs` exactly (scaled-derivative convention, SIMD split path, BC ghost extension). Predicted virtual-node floor at N=512: `φ ≈ 9.1·10⁻¹⁶` (sub-check (e): `‖f¹⁰‖_∞·dx¹⁰/10!·C_w·Λ_M`), **1638× below SepticHermite's 1.49·10⁻¹²**, comfortably under the RELEASE_BLOCKING target ≤ 5·10⁻¹⁴ (`G_OCTONIC_HERMITE_FLOOR`). Condition-number 1-norm sup = 1.42 (≤10 bound). The sampler is mathematically sound and is the correct KEYSTONE lever for the virtual-node floor; it does NOT by itself unblock the ζ⁶/ζ⁸ TRUTHFUL_ORDER pair-slope gates — that depends additionally on the spatial-stencil floor (ADR-0118) and the gate-geometry constraint diagnosed in ADR-0119 (NO-GO).
