# ADR-0120 — `Diffusion6thChernoff` genuine spatial order is 6; v7 gate cluster fixed by floor-free baskets, not kernel change

- **Status**: ACCEPTED 2026-06-05 — empirical floor-free measurement on `feat/v7.0.0-debt-closure`. No kernel/sampler math change warranted; gate cluster recalibrated to floor-free baskets.
- **Decision-maker**: ai-solutions-architect
- **Type**: gate-cluster recalibration verdict (no new types, no math edit). Couples to the v7.0.0 QuinticHermite + `legacy-quintic` REMOVAL (BREAKING).
- **Evidence**: `.dev-docs/v7-diffusion6th-order-verdict.md` (floor-free sweeps + per-gate table).

**Decision.** The −4.5788 g3_6 slope under the SepticHermite default is a **floor artifact, not the genuine spatial order**: for constant `a` the `Diffusion6thChernoff` spatial accuracy comes entirely from K7 off-grid sampling through the grid interpolant (`src/diffusion6.rs:509` zeroes the ζ⁶ τ²-correction), so convergence saturates at the interpolant floor (Septic ≈ 1.5e-12). Floor-free measurement (exact a≡0.5 Gaussian oracle, T=0.5, n=4000) gives the **genuine asymptotic order ≈ 6**: the truncation sub-basket {251,503,997} slopes −6.43 (log₂ ratios 6.09, 6.72), and three recalibrated floor-free baskets — {127,251,503,997} (−6.25), {151,301,601,1201} (−6.42), {127,251,503,997,1499} (−6.53) — all PASS the unchanged ≤ −5.85 gate with the Septic default and **no QuinticHermite anywhere**. QuinticHermite was therefore **not masking a false pass** — its *higher* floor (~1e-10) merely kept the original {…,1999,3989} basket inside the truncation regime, while Septic's *lower* floor pulls those two finest points below the floor and flattens the OLS. Resolution per gate (full table in the verdict note): **RECALIBRATE-BASKET** for the spatial-slope cluster (g3_6, wide_sweep, liouville var-a, doubling) — swap to floor-free baskets + `SepticHermite`, keep all order thresholds; **RECALIBRATE-HONEST** for `g_zeta4_const_a_richardson_ratio` only (temporal order-4; lower gate 3.5→3.1 to the Septic-default re-measurement 3.226, prior 3.5 was QuinticHermite-floor-calibrated per ADR-0086 AMD 1; `_cheb` variant already passes); **TEST-PRECONDITION** for both path_eps gates — the CubicHermite *control* must explicitly set `InterpKind::CubicHermite` (it silently inherits the v6 Septic default → control reads 1.38e-12 and fails its `baseline>1e-7` precondition), then retarget the improved branch QuinticHermite→Septic (1e-8 gate still passes). bug-fixer implements all test recalibrations/preconditions; agentic-engineer owns the coupled QuinticHermite/`legacy-quintic`/`with_quintic_sampling` removal. All gates must then run GREEN under `cargo run -p xtask -- test-ignored-gates` (no `legacy-quintic`). No `triz` contradiction: the gates merely mis-located their measurement window relative to a floor that moved when the default interpolant changed in v6.0.
