# ADR-0141: PyO3 Chunked GIL-Cooperative Evolve (v8.1.0 A-4)

**Status**: Accepted | **Date**: 2026-06-08 | **Author**: agentic-engineer

Adds `Heat1D.evolve_chunked(t, total_steps, chunk_steps, progress=None)` to the PyO3 binding: the Chernoff product runs in chunks of `chunk_steps` steps with `tau = t / total_steps` per step, releasing the GIL via the existing `py.detach` (ADR-0031) for each chunk's pure-Rust compute and re-acquiring it between chunks to (a) call the optional `progress(done, total)` callback and (b) service Python signals such as `KeyboardInterrupt`. The **GIL-held-callback invariant** is strict: the progress callable is never invoked from inside `py.detach`; calling into Python with the GIL dropped is unsound. The **0-ULP parity guarantee** holds because `compute_chunk` applies the Chernoff kernel directly with the same `tau = t / total_steps` — it does not create a new `ChernoffSemigroup(n=chunk_steps)` that would recompute `tau` — so the arithmetic is identical to the monolithic `evolve`. This is **not** a full async runtime: no `pyo3-asyncio`, no `tokio`, no `Future` — just cooperative chunking via the already-established three-phase GIL pattern. A full async runtime remains deferred (ADR-0028).
