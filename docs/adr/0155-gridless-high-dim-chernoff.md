## Context

The single highest-value open problem in computational PDEs is the **curse of dimensionality**: a Cartesian `GridND<F,D>` costs `O(N^d)` time and memory, so `d ≳ 5` is already infeasible (`SmolyakGridND` reaches D=6 before blow-up, `G_SMOLYAK_D6` PASS). The unused resource **R1** is that formula (6) *is itself a Markov transition law*: its adjoint `S*(τ)ρ = ¼τ_{+h}ρ + ¼τ_{−h}ρ + ½τ_{+k}ρ + τc·ρ` is proven mass-conservative (§38.6) and uniformly tight by Prokhorov (§38.5), sympy-certified by `T_ADJOINT_FP_TIGHTNESS` (6/6). The shipped `AdjointFokkerPlanckChernoff<C,F,D>` + `MeasureState<F,D>` (v8.0.0) already iterate this kernel on a sparse-Dirac + Gaussian-background measure — **this is the gridless evolver in embryo**. The `§32 AnisotropicShiftChernoffND` already documents that for `D≥5` the `q^D` tensor quadrature explodes and "falls back to a sparse-grid or Monte-Carlo backend" — the particle evolver is the natural Backend E.

## Decision

Ship a **bounded-dimension (d ≤ ~10) deterministic-branching particle evolver** that iterates one §38 adjoint push-forward step on a `MeasureState<F,D>` ensemble `ρ = Σ wᵢ δ_{xᵢ}` at `O(d·P)` per step, independent of any ambient grid `N`. **TRIZ resolution (research §2.3).** **ТП**: high-`d` accuracy wants a *fine* heat-kernel resolution (⇒ dense `N^d` grid, impossible) AND *cheap* memory (⇒ few points, loses accuracy). **ФП**: the resolution element must be *both fine and coarse*. **ИКР**: the resolution element *itself carries the kernel* — each particle's deterministic ¼,¼,½ 3-way branch *is* the quadrature of its own characteristic flow, rather than sampling a fixed lattice. **ВПР**: the formula's own transition structure (R1), already present, already proven tight. The `4ⁿ` Dirac blow-up is tamed for the **linear-coefficient case** by a **particle-reduction operator** (weighted-Voronoi merge / Gaussian-smoothed background) at a documented, bounded discretization cost (§50.4). The defensible, unique claim: the *deterministic* branching is a **variance-reduced (quasi-MC-like) particle scheme** — which a *random* walk structurally cannot claim, and which this library is uniquely positioned to verify because it already ships the measure-side math (§38) and the d-D shift (§32).

## Consequences

Additive surface: a new `gridless.rs` module building on `adjoint_fp.rs` / `MeasureState`, plus the particle-reduction operator. No existing kernel semantics change. **NARROW ship scope**: linear (constant- and slowly-varying-coefficient) generators, `d ≤ ~10`, **validated envelope d=2**; high-`d` (d>10) nonlinear/variable-coefficient gridless evolution and the general `4ⁿ`-reduction theory are **research-track**, NOT release-blocking (research §5.2). **Strongest failure mode (adversarial, research §2.5):** (1) keeping all branches multiplies particles ~3–4× per step → exponential in trajectory length `n`, *worse* than the grid; any randomized reduction re-introduces variance, so the "lower variance than MC" claim must be **measured, not assumed** — hence `G_GRIDLESS_VARIANCE` is a **go/no-go**. (2) **MLP** (Hutzenthaler-Jentzen et al.) already beats the curse with a *proven* polynomial bound for the semilinear class — so C must win on **memory + determinism + bit-reproducibility** (the H-MEM flagship axis, 0-ULP culture), NOT asymptotic rate; "deterministic but slower" is a niche, not a paradigm shift. Gates: `G_GRIDLESS_DIM_SCALING` (cost-vs-`d` log-log slope ≤1.1, sub-exponential peak memory, sup-error <5e-3 over `d∈{2,4,6,8,10}`); `G_GRIDLESS_VARIANCE` (≥2× variance reduction vs MC random-walk at equal budget P, within a documented factor of Sobol-QMC; **if it fails, C reverts to research-only and Shift B becomes the headline**); `T_GRIDLESS_*` sympy oracle (one-step particle push-forward == §38 adjoint kernel, reusing `T_ADJOINT_FP_TIGHTNESS` machinery). High-d research-track direction: path-space RQMC functional estimation (ADR-0158).

## Amendment 1 (v9.0.0 implementation outcome) — 2026-06-09

The pre-registered §50.6 go/no-go fired as designed. Measurements under a fair shared-CRN comparison protocol (commit `f633cde`; symmetrized per-axis sweep, d-dimensional product-bin reduction, equal-budget CRN quantization of δ₀):

- `T_GRIDLESS_*` sympy oracle: **PASS 3/3** (push_forward_exactness / mass_conservation / voronoi_moment_match).
- `G_GRIDLESS_DIM_SCALING`: **d=2 PASS** (accuracy err 1.197e-3 < 5e-3 gate). d=3 requires 8× cap increase; d≥4 accuracy collapses (err 9.75e-2 … 7.05e-1). Root cause: spatial-merge particle reduction (`R_P`) needs `cap ≈ m^d` to hold `m` bins/axis — the curse of dimensionality re-enters through the reduction grid. This is an INTRINSIC LIMIT of the particle representation, not a tunable implementation bug (§50.7).
- `G_GRIDLESS_VARIANCE`: **NO-GO** — at d=2, deterministic branching beats MC by MSE ratio 1.417× (fair CRN comparison), below the pre-registered ≥2× gate. Low-variance thesis not confirmed.

**Decision (§50.6 consequence, pre-registered):** `GridlessChernoff` ships **NARROW** — d=2 validated deterministic d-dim primitive, retained as a correct library component and the documented negative result. High-dimensional gridless via path-space RQMC remains research-track (ADR-0158 PROPOSED). Reporting this negative result as the designed honest outcome of the pre-registered gate.

## Amendment 2 (Shift C RESOLUTION — tensor-train carrier) — 2026-06-10

The INTRINSIC LIMIT established in Amendment 1 is not a failure of the Chernoff evolver — it is a failure of the particle representation as the state carrier. The curse of dimensionality re-enters through the spatial-merge reduction grid because the Chernoff branching is a deterministic quadrature (error = bias) and reducing bias in d dimensions in a **particle** representation is `O(m^d)`. The resolution: replace the particle carrier with a **low-rank tensor-train (TT)** carrier.

**ACCEPTED: `TtChernoff` (ADR-0159, §52)** — the per-axis Chernoff shift is an O(1)-rank, single-mode, rank-preserving TT-operator (Kazeev–Khoromskij, Grade A); for the linear diagonal-A (Gaussian) class the TT-rank is algebraically capped at r ≤ d/2 (Rohrbach–Dolgov–Grasedyck–Scheichl, Grade A); storage is `O(d³·n)` — polynomial in d, exponential curse escaped. The rank-1 case reduces exactly to the shipped Strang⊗ tensor product (§52.3). Gate `G_TT_CHERNOFF_DIMSCALING` (PRE-REGISTERED, `slow-tests`). Honest scope: diagonal-A Gaussian class only; off-diagonal/variable/nonlinear = research-track.

**Combined Shift C verdict (v9.0.0 FINAL):** `GridlessChernoff` = d=2 particle primitive + negative result (Amendment 1, §50). `TtChernoff` = working curse-escaping form for the diagonal-A Gaussian class (Amendment 2, §52, ADR-0159). Both are correct, both are normative, and both are retained. The two are complementary: §50 documents what was tried and refuted under the particle representation; §52/ADR-0159 documents the escape that works by changing the representation entirely.
