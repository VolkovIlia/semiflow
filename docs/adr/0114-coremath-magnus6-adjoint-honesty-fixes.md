# ADR-0114 — Core-math honesty fixes: Magnus K=6 order collapse + Adjoint scalar fudge

- **Status**: Proposed (design/spec; implementation pending — see `.dev-docs/reports/bug-a-b-coremath-fixspec-2026-05-31.md`)
- **Date**: 2026-05-31
- **SemVer**: BREAKING (numerics + API) → **v7.0.0 candidate**
- **Supersedes/amends**: ADR-0055 (AdjointChernoff), ADR-0056 (Magnus K=6); math.md §15.1/§15.1.bis and §16.2 are NORMATIVE-WRONG and must be corrected.

## Decision

**BUG-A (Magnus K=6, `magnus6_graph.rs`)** — verdict FORMULA_WRONG + CODE_BUG + TEST_MASK. The Ω₆ in math.md §16.2 and the code header is *not* an order-6 Magnus integrator: independent sympy/scipy re-derivation on a genuinely non-commuting `A(t)` measures global order ~2-3 (local slope ~3, vs the ~7 required), and the hand-assembled nested commutator (line 622) differs from the true `[A₂,[A₃,A₁]]` by `−2·L₂·[A₃,A₁]`. The single order-6 gate (G17) used a *uniform scalar* edge weight `w(t)=1+0.3sin(πt)` making all `L(t)` commute, so every commutator term was zero and the bug was masked. Replace Ω₆ with the canonical Blanes-Casas-Oteo-Ros (Phys.Rep.470, 2009) 2-commutator form using scaled GL₆ generators `B₁=τA₂`, `B₂=τ(√15/3)(A₃−A₁)`, `B₃=τ(10/3)(A₃−2A₂+A₁)`, which is numerically verified to local order 7 (global 6); replace G17 with a *distinct-per-edge-phase* family `wᵢ(t)=1+0.3sin(π(t+φᵢ))` (proven non-commuting, with a non-kernel IC and an in-test non-commutativity precondition assertion so it cannot be re-masked); also document/guard the latent `apply_into` `t_start=0` hardcode (correct only for autonomous `L`).

**BUG-B (Adjoint, `adjoint.rs`)** — verdict RATIONALIZED_FUDGE + DESIGN_GAP + TEST_MASK. The "dual correction" `dst += −τ·(‖src‖²/n)·src` is generator-independent, nonlinear in `g`, and is *not* the `−τ·proj_B` of math.md §15; it violates `⟨Su,g⟩=⟨u,S*g⟩` by up to ~96% on a drifted inner (independently reproduced), while the true `exp(τAᵀ)` satisfies it to machine precision. A correct generic adjoint is structurally impossible from `apply_into`+`dot`+`axpy` alone (no access to `Aᵀ`'s action). Therefore: keep `new_self_adjoint` (correct by delegation); make `new_general` *refuse* — return `SemiflowError::UnsupportedOperation` rather than emit silent wrong numbers — until an opt-in `AdjointApply::apply_adjoint_into` primitive lands (implemented first for `DriftReactionChernoff`, whose transpose is the same kernel with negated drift). Replace the masking G16c gate (which measured the real residual but asserted only `< 1.0` with an "O(τ) by design" rationalization) with G16c′: assert either the honest `UnsupportedOperation` error or, once the primitive exists, a tight order-tied tolerance against an independent `exp(τAᵀ)` oracle.

## Rationale

Both bugs share one anti-pattern: a gate engineered around the wrong number (commuting family / `< 1.0` threshold) lets an incorrect implementation report success. The honest fixes (a) replace the wrong formula with a literature-verified one, (b) make impossible operations fail loudly instead of fudging, and (c) replace masking gates with ones that assert the *defining* property (genuine non-commutativity for order; the adjoint identity itself) so re-masking is structurally prevented. gitnexus impact: both kernels are internal (`apply_omega6` HIGH-but-internal; `apply_dual_evolution` LOW); numeric change to Magnus K=6 ripples to semiflow-py/ffi/wasm bindings (BREAKING), the adjoint API change is contained to Rust callers of `new_general`.
