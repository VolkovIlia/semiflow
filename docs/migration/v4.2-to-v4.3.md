# Migration Guide: v4.2 → v4.3 (Chebyshev spectral collocation + ζ⁸ Wave II)

**ADR-0090 / ADR-0088 Wave II** — Chebyshev spectral collocation (M=64 virtual Lobatto
nodes) is available on all 1D kernels via opt-in builder methods. A new ζ⁸ kernel
(`Diffusion8thZeta8Chernoff`) ships with Chebyshev **default ON** (order-8 contract
requires the spectral spatial floor).

## What changed

| Kernel | v4.2 default | v4.3 default | Opt-in |
|--------|-------------|-------------|--------|
| `Diffusion4thChernoff` (K5) | CubicHermite | **unchanged** | `.with_chebyshev_sampling()` |
| `Diffusion4thZeta4Chernoff` (ζ⁴) | CubicHermite | **unchanged** | `.with_chebyshev_sampling()` |
| `Diffusion6thZeta6Chernoff` (ζ⁶) | QuinticHermite | **unchanged** | `.with_chebyshev_sampling()` |
| `Diffusion8thZeta8Chernoff` (ζ⁸) | N/A (new) | **ChebyshevSpectral M=64** | `.without_chebyshev_sampling()` |

**ζ⁴ and ζ⁶ default unchanged from v4.2**: v4.2 byte-equality preserved for existing code.
Chebyshev is a strict opt-in on ζ⁴/ζ⁶ (ADR-0090 §"Backward compat").

**ζ⁸ defaults Chebyshev ON**: order-8 contract requires spectral floor ≤ 1e-15.
QuinticHermite floor (~1e-10) causes catastrophic K=4 Richardson stagnation
(ADR-0089 AMENDMENT 1 Insight #5, ADR-0088 AMENDMENT 1).

## New: `InterpKind::ChebyshevSpectral { m: usize }`

The `InterpKind` enum gains a new variant:

```rust
pub enum InterpKind {
    CubicHermite,          // default (v4.2 behavior preserved)
    QuinticHermite,        // opt-in (ADR-0089)
    ChebyshevSpectral { m: usize }, // NEW: ADR-0090, f64-only
}
```

The `m` field is the number of Chebyshev-Lobatto intervals (M+1 virtual nodes).
Supported values: `{8, 16, 32, 64, 128, 256, 512}`.

**f64-only restriction**: `ChebyshevSpectral` is only supported for `Grid1D<f64>`.
Non-f64 callers receive `SemiflowError::Unsupported { feature: "ChebyshevSpectral is only supported for Grid1D<f64>" }`.

## No breaking changes for existing ζ⁴/ζ⁶ users

`Diffusion4thZeta4Chernoff::new()` and `Diffusion6thZeta6Chernoff::new()` have
identical defaults to v4.2. No `Unsupported` error unless you explicitly pass
a `ChebyshevSpectral` grid with a non-f64 type.

## Opt-in to Chebyshev on K5/ζ⁴/ζ⁶

```rust
use semiflow_core::{
    Diffusion4thChernoff, Diffusion4thZeta4Chernoff,
    Diffusion6thZeta6Chernoff, Grid1D,
};

let grid = Grid1D::new(-10.0_f64, 10.0, 512).unwrap();
let k5 = Diffusion4thChernoff::new(|_| 1.0, |_| 0.0, |_| 0.0, 1.0, grid);

// Default CubicHermite (v4.2 byte-identical)
let k5_cubic = k5.clone();

// Opt-in to Chebyshev M=64 on K5:
let k5_cheb = k5.clone().with_chebyshev_sampling();

// Opt-in with custom M (e.g., M=128 for higher-resolution spectral floor):
let k5_cheb_128 = k5.clone().with_chebyshev_sampling_m(128);

// Propagates through ζ⁴:
let zeta4 = Diffusion4thZeta4Chernoff::new(k5.clone(), Some(1.0)).unwrap();
let zeta4_cheb = zeta4.with_chebyshev_sampling();  // ζ⁴ + Chebyshev

// Propagates through ζ⁶:
let zeta4_for_zeta6 = Diffusion4thZeta4Chernoff::new(k5.clone(), Some(1.0)).unwrap();
let zeta6 = Diffusion6thZeta6Chernoff::new(zeta4_for_zeta6, Some(1.0)).unwrap();
let zeta6_cheb = zeta6.with_chebyshev_sampling();  // ζ⁶ + Chebyshev
```

## New: `Diffusion8thZeta8Chernoff` (ζ⁸, order-8 temporal)

```rust
use semiflow_core::{
    Diffusion4thChernoff, Diffusion4thZeta4Chernoff,
    Diffusion6thZeta6Chernoff, Diffusion8thZeta8Chernoff, Grid1D,
};

let grid = Grid1D::new(-10.0_f64, 10.0, 512).unwrap();
let k5 = Diffusion4thChernoff::new(|_| 1.0, |_| 0.0, |_| 0.0, 1.5, grid);
let zeta4 = Diffusion4thZeta4Chernoff::new(k5, Some(1.5)).unwrap();
let zeta6 = Diffusion6thZeta6Chernoff::new(zeta4, Some(1.5)).unwrap();

// ζ⁸ defaults Chebyshev M=64 ON (required for order-8 contract):
let zeta8 = Diffusion8thZeta8Chernoff::new(zeta6, Some(1.5)).unwrap();
assert_eq!(zeta8.order(), 8);

// Downgrade to Quintic (debugging only — G_zeta8 gates WILL fail):
let zeta8_debug = zeta8.without_chebyshev_sampling();

// Use higher M (e.g., for tighter spectral floor):
let zeta8_m128 = Diffusion8thZeta8Chernoff::new(
    Diffusion6thZeta6Chernoff::new(
        Diffusion4thZeta4Chernoff::new(
            Diffusion4thChernoff::new(|_| 1.0, |_| 0.0, |_| 0.0, 1.5, grid),
            Some(1.5),
        ).unwrap(),
        Some(1.5),
    ).unwrap(),
    Some(1.5),
).unwrap().with_chebyshev_sampling_m(128);
```

### Caller invariants for ζ⁸

1. `f ∈ D(A^8)`: pre-check `kernel.in_subspace::<8>(&f)` once.
2. `a ∈ C^8_b`: assert via `a_kth_bound: Some(c)`.
3. `a(x) > 0` everywhere (strict ellipticity, inherited from inner).
4. Grid ≥ 33 points (required for `in_subspace` to return `true`).

### Work per step

Each outer step makes **3 inner R³ (ζ⁶) calls** = **27 K5 base evaluations**:

```text
coarse = R³(τ)   · src   // 9 K5 calls
half   = R³(τ/2) · src   // 9 K5 calls
fine   = R³(τ/2) · half  // 9 K5 calls
dst    = (64·fine − coarse) / 63
```

## Checklist for upgrading to v4.3

- [ ] Existing ζ⁴/ζ⁶ code: no changes needed (defaults unchanged, byte-identical).
- [ ] If you use `InterpKind` explicitly: add a match arm for `ChebyshevSpectral { m }`.
- [ ] New ζ⁸ users: provide `a_kth_bound: Some(c)` for `in_subspace` to work.
- [ ] f32 users: Chebyshev is f64-only; f32 kernels are unaffected.

## Acceptance gates (RELEASE_BLOCKING)

- `G_PATH_EPS_CHEB_FLOOR` — Chebyshev floor ≤ 1e-8 at N=512 Gaussian probe.
- `G_zeta8_const_a_richardson_cheb` — ζ⁸ Richardson ratio log₂ ≥ 6.5.
