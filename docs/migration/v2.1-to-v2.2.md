# Migration Guide: semiflow-core v2.1 → v2.2

**Scope**: v2.1.0 → v2.2.0 (ADR-0052–0060, 2026-05-21).
**Most users need ZERO code changes.** v2.2 is a MINOR release — all
public API surfaces from v2.0 and v2.1 remain source-compatible.

---

## TL;DR — Decision tree

```
Using NonSeparable2DChernoff or NonSeparable2DAnisotropicChernoff?
  → Type aliases are in place; your code compiles unchanged. §5.

Using GraphHeatChernoff with a fixed topology?
  → No change required. §2 shows the new time-varying path for reference.

Want adjoint / backward semigroup?
  → New: AdjointChernoff<C, F>. §3.

Want to solve Schrödinger?
  → New: SchrodingerChernoff<F> + SchrodingerState<F>. §4.

Want graph kernels from C / Python / JavaScript?
  → New graph bindings in all three crates. §6.

Want to run thread-scaling benchmarks?
  → cargo run -p xtask -- bench-parallel. §7.
```

---

## §1 — What's new in v2.2

v2.2 closes 8 items deferred from v2.0 and v2.1 across three coordinated
waves plus a parallel benchmark phase.

**Wave A** adds three types for time-varying graph diffusion: `GraphTraj<F>`
models a piecewise-smooth sequence of graph topologies; `VarCoefGraphHeatChernoff<F>`
extends the graph heat kernel to variable node coefficients with ζ-A correction;
`MagnusGraphHeatChernoff::evolve_with_traj` drives trajectory-aware Magnus K=4
evolution across topology-change breakpoints.

**Wave B** adds three advanced semigroup types: `AdjointChernoff<C, F>` wraps
any `ChernoffFunction` to produce its adjoint (backward) semigroup;
`MagnusGraphHeat6thChernoff<F>` provides order-6 Magnus on time-dependent graph
Laplacians (f64 only); `SchrodingerChernoff<F>` + `SchrodingerState<F>` solve
the real-representation Schrödinger equation via palindromic Strang splitting.

**Wave C** unifies the `NonSeparable2DChernoff` (v0.7, scalar coupling) and
`NonSeparable2DAnisotropicChernoff` (v0.9, anisotropic β) pair under the new
`NonSeparableMixedChernoff<X, Y, F, S>` generic — old names remain as type
aliases. Graph kernels are now exposed in all three binding crates (C, Python,
WASM).

**Phase 5** ships a Criterion thread-scaling benchmark suite
(`cargo run -p xtask -- bench-parallel`). Results are in
`docs/perf/scaling-v2_2_0.md`.

**Zero breaking changes.** `cargo semver-checks` PASS confirmed locally.

## §2 — Graph PDE: GraphTraj and time-varying topology

v2.1 callers using `GraphHeatChernoff` or `MagnusGraphHeatChernoff` with a
fixed topology continue to work without any changes.

To migrate a fixed-topology v2.1 call to the new trajectory API:

```rust
// v2.1 — fixed topology, using LaplacianAtTime<F>
let laplacian_at: LaplacianAtTime<f64> = Box::new(move |_t| Arc::clone(&laplacian));
let magnus = MagnusGraphHeatChernoff::new(
    Arc::clone(&graph), laplacian_at, rho_bar_max, check_radius,
);
let u1 = ChernoffSemigroup::new(magnus, n_steps)?.evolve(t_final, &u0)?;
```

```rust
// v2.2 — same fixed topology expressed as a single-segment GraphTraj
use semiflow_core::{GraphTraj, SegmentWeightFn};

let l_arc = Arc::clone(&laplacian);
let traj = GraphTraj::new(vec![
    (0.0, Arc::clone(&graph), Box::new(move |_t| Arc::clone(&l_arc)) as SegmentWeightFn<f64>),
]).expect("single-segment trajectory is always valid");

let magnus = MagnusGraphHeatChernoff::new(
    Arc::clone(&graph),
    Box::new(|t| traj.laplacian_at(t)),
    rho_bar_max,
    check_radius,
);
let u1 = magnus.evolve_with_traj(&traj, t_final, n_steps, &u0)?;
```

For the common fixed-topology case, the convenience constructor
`GraphTraj::fixed_topology(graph, laplacian)` produces a single-segment
trajectory and can replace the three-line `vec!` construction above.

**Key contracts for `GraphTraj`**:
- Each segment's `SegmentWeightFn` must be C² in `t` within the segment
  (GL4 quadrature error analysis requires this).
- Topology (`row_ptr`/`col_idx`) MUST NOT change within a segment. Topology
  changes only happen at breakpoints. The library asserts this in debug mode.
- Maximum segments: `MAX_GRAPH_TRAJ_SEGMENTS = 65_535`.

## §3 — Adjoint wrapper: AdjointChernoff

`AdjointChernoff<C, F>` wraps any `C: ChernoffFunction<F>` where
`C::S: HilbertState<F>` and produces the adjoint (backward) semigroup.

```rust
use semiflow_core::{AdjointChernoff, GraphHeatChernoff, ChernoffSemigroup};

// Inner forward kernel
let inner = GraphHeatChernoff::new(Arc::clone(&laplacian));

// Self-adjoint case (the graph heat kernel is self-adjoint for symmetric L_G)
let adjoint = AdjointChernoff::new_self_adjoint(inner);
// adjoint.order() == inner.order()  (self-adjoint preserves order)

// Non-self-adjoint case (e.g. drift-reaction kernel)
let adjoint_ns = AdjointChernoff::new_general(inner_ns);
// adjoint_ns.order() == min(inner_ns.order(), 2)

let semi = ChernoffSemigroup::new(adjoint, n_steps)?;
let u_adj = semi.evolve(t_final, &u0)?;
```

The `is_self_adjoint` flag triggers the 0-ULP identity path (G15 gate). For
the general case, a dual-pairing O(τ) correction is applied. Use
`AdjointChernoff::detect_self_adjointness(&kernel, &signal, tol)` as a
probabilistic developer-time check (not a runtime invariant).

## §4 — Schrödinger basics and Crank-Nicolson note

`SchrodingerChernoff<F>` solves `i ψ_t = (−∂_xx + V(x)) ψ` using the
real-only `(ψ_re, ψ_im)` representation (ADR-0057 Option A). The kinetic
step `K(τ)` uses `Diffusion4thChernoff` (4th-order spatial accuracy).

```rust
use semiflow_core::{
    Grid1D, SchrodingerChernoff, SchrodingerState, ChernoffSemigroup,
};

let grid = Grid1D::new(-10.0, 10.0, 512).expect("grid OK");
let kernel = SchrodingerChernoff::new(|x: f64| x * x / 2.0, grid);
let semi = ChernoffSemigroup::new(kernel, 200)?;

let psi0 = SchrodingerState::from_real(
    grid,
    |x| (-x * x / 2.0).exp(),  // ψ_re
);
let psi1 = semi.evolve(1.0, &psi0)?;

// Unitarity check: ‖ψ₁‖² ≈ ‖ψ₀‖² (G18 gate: < 1e-12 for f64)
let norm0 = psi0.norm_sq_total();
let norm1 = psi1.norm_sq_total();
assert!((norm1 - norm0).abs() < 1e-10);
```

**Crank-Nicolson kinetic note (ADR-0057 Amendment 1)**: the Crank-Nicolson
half-step variant of the kinetic operator was deferred to v2.3. In v2.2, the
kinetic step uses `Diffusion4thChernoff` throughout. If you need a
Crank-Nicolson kinetic step, track ADR-0057 Amendment 2 (planned for v2.3).

`SchrodingerState<F>` implements `HilbertState<F>`. The norm methods work on
the combined `(ψ_re, ψ_im)` 2N-dimensional vector.

## §5 — NonSeparable refactor: your existing code still works

`NonSeparable2DChernoff<X, Y>` (v0.7.0) and `NonSeparable2DAnisotropicChernoff<X, Y>`
(v0.9.0) are now type aliases for `NonSeparableMixedChernoff<X, Y, F, S>`
specialisations. All existing call sites compile unchanged.

```rust
// v2.1 — still compiles in v2.2 (type alias)
let ns: NonSeparable2DChernoff<_, _> =
    NonSeparable2DChernoff::new(cx, cy, |_, _| 0.3_f64, grid2d);

// v2.2 — explicit generic form (same behavior)
let ns: NonSeparableMixedChernoff<_, _, f64> =
    NonSeparableMixedChernoff::new_scalar(cx, cy, |_, _| 0.3_f64, grid2d);
```

The G20 alias-identity gate confirms 0 ULP difference between the alias path
and the generic path for both v0.7 (scalar c) and v0.9 (anisotropic β) inputs.

ADR-0033 (which had originally planned to keep both types as first-class API)
is now SUPERSEDED by ADR-0058. The type aliases mean no deprecation cycle is
needed.

## §6 — Graph bindings (FFI / PyO3 / WASM)

Graph PDE kernels are now exposed in all three binding crates. See the binding
crate README files for full API reference and build instructions.

**C (semiflow-ffi)**:

```c
#include "remizov.h"

RemizovGraphHeat *gh = remizov_graph_heat_new_path(100 /*n_nodes*/, 1 /*n_steps*/);
double *u0 = /* ... initial signal ... */;
double *u1 = malloc(100 * sizeof(double));
remizov_graph_heat_evolve(gh, 0.5 /*t*/, u0, u1);
remizov_graph_heat_free(gh);
```

**Python (semiflow-py)**:

```python
from semiflow import GraphHeat

gh = GraphHeat.path(100, n_steps=500)  # 100-node path graph
u0 = [math.exp(-(i - 50.0)**2 / 32.0) for i in range(100)]
u1 = gh.evolve(0.5, u0)
```

**JavaScript/WASM (@semiflow/wasm)**:

```js
import init, { GraphHeat } from "@semiflow/wasm";
await init();
const gh = GraphHeat.path(100, 500);
const u1 = gh.evolve(0.5, u0Float64Array);
```

Cross-binding sup-error is ≤ 3 ULP (ADR-0059). See
[`docs/adr/0059-graph-bindings.md`](../adr/0059-graph-bindings.md) for the
full specification and the `G_cross_binding_graph_identity` CI gate.

## §7 — Parallel benchmarks

v2.2 adds a Criterion thread-scaling benchmark suite. To run it:

```bash
cargo run -p xtask -- bench-parallel
# Equivalent to:
# RUSTFLAGS="-C target-cpu=native" cargo bench -p semiflow-core \
#   --features parallel,simd \
#   --bench strang2d_parallel --bench strang3d_parallel \
#   --bench ns2d_aniso_parallel --bench graph_heat
```

Results are written to `docs/perf/scaling-v2_2_0.md`. The baseline JSON is at
`docs/perf/baseline-v2_2_0.json`. Key findings on i7-12700K (12C/20T):

| Suite        | S(8)  | η(8)  | Note |
|--------------|------:|------:|------|
| Strang2D     | 5.07× | 0.634 | Memory-bandwidth limited (> L3) |
| Strang3D     | 5.58× | 0.698 | Memory-bandwidth limited (> L3) |
| NS2D-aniso   | 1.01× | 0.126 | INTRINSIC_LIMIT: serial AxisLift path |

NS2D-aniso parallelism requires a parallel `AxisLift` impl, planned for v2.3.

## §8 — Breaking changes

**There are none.** v2.2.0 is a MINOR release under v2.x.

The `NonSeparable2DChernoff` and `NonSeparable2DAnisotropicChernoff` types
are preserved as type aliases (not removed or deprecated). All trait impls,
method signatures, and `ChernoffFunction` contracts from v2.1 remain
source-compatible.

`cargo semver-checks --baseline-rev v2.1.0 --release-type minor --workspace`
exits 0.
